Interesting facts noted during classes:

  • Avoid global variables such as hectic 'var'. Tip: use 'let' instead, or alternativelly, when you know your code is not going to change, go for 'const'.
  • Global access modifiers are also to be avoided in most cases. The notations is '__name'.
  • Hoisting - you can give values to variables and define fuctions later as JS reads whole code, parses it and eventually looks for suitable functions to run.
  • A Callback is a feature that is Node.js's asynchronous nature related. This is simply a function passed as an argument to another function which will then use it (call it back).
  • Callback function reduces waiting time as it prevents any blocking when a proces takes a little bit more time, and allows other code to be run in the meantime.
  • Arrow notation is better that dot notation when it comes to a 'create new function' syntax. It saves some code, therefore makes your syntax shorter, simpler and thus makes your code readable.
  • When you send your code to somebody, either by zip file or by pushing to github, always remember not to include node_modules as they're heavy and can cause merge conflicts. Tips add node_modules folder to the .gitignore file.
  • GET request is unencrypted! You should never use it with sensitive data. Tip: use encrypted POST instead.
  • In GET request, dynamic variables are defined after collon.
  • To avoid so called "header" error, use return statement before res.send();
  • /script is called SELF closing tag, not just closing tag.
  • Link is called anchor tag in JS.
  • Double-quoted strings are preferred by the jQuery style guide, though single-quoted strings are often used as well.
